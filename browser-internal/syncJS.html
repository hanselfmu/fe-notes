It's again about the timing of the first paint, and the heuristic behind it. If the DOM to be painted is large (in this case ~1000 DOM elements), the first paint will happen before the first script tag. If the DOM elements are only a few, the first paint will happen after the first script tag.
This only applies to inline scripts; external scripts are always processed AFTER first paint. This is why synchronous external JS scripts should be put at the bottom.
